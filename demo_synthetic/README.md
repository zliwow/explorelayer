# demo_synthetic — Qwen port of the Claude ground-truth demo

Vendored from `gdsattemptclaude/`. Same synthetic analog mini-chip, same
detector, but the narrative panel is generated by a local Qwen/sglang
server instead of being hardcoded from Claude.

## Run

```
python make_demo_gds.py
python find_issues.py demo_chip.gds --config layer_config.json --output findings.json
python render_report.py
```

Output: `demo_report.png`.

`render_report.py` hits `http://localhost:8000/v1/chat/completions` by
default and auto-discovers the model id from `/v1/models`. Override
with `--llm-url` / `--llm-model`. If the server isn't reachable the
narrative falls back to the original Claude-era prose so the render
never fails end-to-end.

## What it proves

Planted ground-truth issue: RDL strip crossing the matched Q1/Q2 BJT
pair in BANDGAP_REF. The detector must surface it as critical, and the
LLM must explain *why* (thermal asymmetry, piezo-stress, capacitive
coupling) — the reasoning quality that separates a credible AI review
from geometric noise.

Compare against `../reference_demo_report.png` (Claude-authored) to
judge whether a 27B-class local model produces comparable depth on a
chip where we know the right answer.
